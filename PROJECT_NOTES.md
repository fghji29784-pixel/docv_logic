## 1. 개요
- **목표**: 활성화공정 OCV 기반 저전압(고자가방전) 불량 선별 로직 고도화 — 과검/미검 동시 저감
- **현재 운영 로직(Baseline)**: 트레이별 (전용OCV1 − 전용OCV3)의 **mode + 고정 오프셋 0.8mV** 단일 임계값
- **코드 저장소**: GitHub `fghji29784-pixel/docv_logic` = 로컬 `C:\Users\fghji\Desktop\docv_logic_repo`
  - `dOCV_analyzer.py`: 6개 방법 비교 분석기 (GUI/CLI, 대시보드 PNG + 결과 Excel 3시트)
  - `dOCV_temp_diagnostic.py`: 온도-OCV 진단 (5패널 PNG + 자동 해석, dU/dT 판정용)
  - `requirements.txt`
  - 환경: **Python 3.8 32bit → scipy/sklearn 설치 불가**, numpy로 전부 대체 구현. 64bit 전환 시 sklearn(GPR/IsolationForest/Huber) 사용 가능

## 2. 데이터 구조 & 라벨 (★주의)
- 144셀/트레이, 12×12 격자. **Cell No 1~144 = A01~L12** (col=(n−1)//12+1=A~L열, row=(n−1)%12+1=1~12행). 한 랏 ~703 트레이
- 컬럼: TRAY ID, Cell No, 전용OCV1~3(**mV**), T1~3(**°C**, 각 OCV 측정시점 셀온도), Label(**A=양품, E=불량**)
- 측정 간격: OCV1→2 ≈1일, OCV2→3 ≈2일 (코드 기본값; 실제값 확인 필요)
- **★Label은 현 Baseline(mode+0.8mV)으로 만든 것 → 순환논리.** 따라서 출력되는 감도/특이도/미검/과검은 "진짜 정확도"가 아니라 **"Baseline과의 일치/차이"**일 뿐. 진짜 평가는 분해/자가방전 재측정한 물리 라벨 필요

## 3. 실제 OCV/ΔOCV 분포 (22°C 필터 후 분석대상 99,293셀 / 제외 1,821셀, 703트레이)
- **ΔOCV(=OCV1−OCV3) 분포**: median ≈ **1.36 mV**, mode ≈ **1.28 mV**, 산포 좁음(대부분 1.0~1.8mV)
- **임계값 비교(이 랏)**: DPAT(median+3.5·MAD) ≈ 1.9mV < **Baseline(mode+0.8) ≈ 2.08mV** → 이 랏에선 **Baseline이 오히려 더 느슨**(고정오프셋의 경직성 실증)
- **K값(3점 기울기)**: median ≈ **0.42 mV/day**, DPAT 임계값 ≈ 0.586
- mode는 분해능 양자화로 불안정: 22°C 필터로 표본 약간 바뀌자 Baseline 판정 **16셀 뒤집힘**(자기 라벨에도 미검5·과검11)

## 4. 6개 방법 + 실제 판정 결과
판정수(분석대상 기준) 및 vs-Baseline라벨 성능:

| 방법 | 판정수(불량률) | 감도 | 특이도 | 미검 | 과검 | 비고 |
|------|---------------|------|--------|------|------|------|
| Baseline | 93 (0.09%) | 0.943 | 1.000 | 5 | 11 | mode+0.8mV, 자기라벨에도 불일치(mode 불안정) |
| C_DPAT | 152 (0.15%) | 1.000 | 0.999 | 0 | 65 | median+k·MAD, **채택권고** |
| A_TempReg | 151 (0.15%) | 1.000 | 0.999 | 0 | 64 | ΔOCV~T 아렌니우스회귀, 온도핀불신→보류 |
| B_NNR | 146 (0.15%) | 1.000 | 0.999 | 0 | 59 | 3×3 이웃중앙값 잔차, **주력** |
| D_Kslope | 16 (0.02%) | 0.092 | 1.000 | 79 | 8 | ΔOCV와 직교(급락 희석)→드리프트모드 검출용 별도게이트 |
| AB_Combined | 156 (0.16%) | 1.000 | 0.999 | 0 | 69 | A∪B |
| E_ML | 5540 (5.58%) | 1.000 | 0.945 | 0 | 5453 | **contamination=0.05 아티팩트**, 실불량률(~0.002)로 재튜닝 필요 |

- C/A/B가 **Baseline 불량 100% 포함 + 추가 ~60셀**(0.06%) 검출. Jaccard에서 Baseline·C·A·B·AB는 초록(높은 일치), D_Kslope·E_ML만 직교
- 방법 정의: Baseline(mode+0.8) / C_DPAT(median+k·1.4826·MAD, k≈3.5, 단측, 바닥값floor 병행) / A_TempReg(ln(ΔOCV)~1/T_K 아렌니우스 지수회귀 잔차에 DPAT) / B_NNR(트레이 내 3×3 이웃 중앙값 대비 잔차에 DPAT) / D_Kslope(3점 직선 기울기 K값에 DPAT + 구간 일관성) / E_ML(IF+LOF 또는 numpy 로버스트 Z, 트레이별)
- 전 방법 **TRAY ID 기준 트레이별 상대판정**(트레이마다 mode/median/MAD/회귀/이웃 독립 계산)

## 5. ★온도 데이터 실측 진단 (핵심)
**5.1 시점 간 온도차 (dU/dT 판정용)**
- T1−T3 **std(로버스트, IQR/1.349) = 0.371°C**, std(전체)=1.154°C(이상치로 부풀려짐), |T1−T3|>1°C = **4.1%**
- **ΔOCV ~ (T1−T3) 기울기 = −0.071 mV/°C** (작고 음수) → 96% 셀에서 보정량 <0.03mV
- 전용OCV 진행에 따라 셀온도 **점진적 하강**(T1>T2>T3, 에이징 열 식음). 단 크기 작음
- 전용OCV3에서 온도 산포 감소(평형 근접) → OCV3 기반 특징이 상대적으로 깨끗

**5.2 공간(트레이 내) 온도**
- 절대온도: 정상핀 거의 **균일 ~25°C(트레이 내 ~0.7°C 폭)**. T_avg 전체범위 17.2°C는 불량핀이 지배
- **불량핀(온도 읽기 오류) ≈ 2.84%**(T_avg가 중앙값−4·robust_std 미만, <24.7°C), 트레이당 ~4개. 히트맵상 고립된 셀(예: 트레이 CFDD000005의 A6·G8·J8). **전반적 노이즈 아님, 드문 불량핀**
- 공간 매끄러움 지표 = 0.47 (대체로 매끄러움+불량핀이 끌어올림)
- **냉각 패턴(온도변화량 T1−T3)이 트레이마다 제각각**(어떤 트레이는 위로, 어떤 트레이는 왼쪽으로 쏠림). 여러 트레이 평균하면 중앙高로 보이나 **개별 트레이는 비일관 → 위치 기반/전역 패턴 보정 불가**(원 문서의 "위치 보정도 통하지 않는다" 실증)

**5.3 E등급-온도 상관 (★해석 주의)**
- 사용자 슬라이드: OCV3에서 E등급 셀 온도가 박스플롯 밖(아래, 차가움)에 다수
- **이는 인과("차가워서 불량") 아님.** Label이 ΔOCV 기준이고 ΔOCV가 온도와 상관(−0.071)이라 E등급이 온도극단에 자동으로 몰리는 **순환/상관**. 오히려 **현 선별이 온도 오염(차가운 셀 과검) 가능성**의 증거. 인자화하려면 물리검증 라벨 필수

## 6. 진단 결론 (온도의 역할)
- **온도는 ΔOCV "보정 입력"으로 불필요(이 랏)**: ① dU/dT 가역보정 효과 미미(<0.03mV)→스킵 확정, ② 트레이 내 절대온도 균일→아렌니우스 보정 헤드룸 작음(A_TempReg≈C_DPAT 실증), ③ 냉각패턴 트레이별 제각각→위치/전역 보정 불가
- **온도 영향은 실재하나 B_NNR(이웃 비교)이 온도 없이 국소 상쇄**가 가장 강건(트레이 형상 방향 무관, 불량핀·온도핀 불신에 둔감)
- **온도 데이터의 진짜 가치 = 보정이 아니라 진단**: ① 불량핀 색출, ② 현 선별의 온도 오염(과검) 증거 제시
- **단서**: 본 결론은 온도핀 부실 + 트레이 내 균일한 현 조건 한정. 온도핀 정밀화 / 실제 강한 공간구배 있는 다른 랏에선 온도 직접보정 가치 상승 가능

## 7. 권고 방향
- **채택**: C_DPAT(임계값 안정화) + B_NNR(공간 강건성). 둘 다 로버스트 통계(median/MAD). 운영은 단변량 룰 1차 + 보조 2차 단계구조
- **별도 게이트**: D_Kslope(완만 드리프트 불량 — ΔOCV가 놓치는 직교 모드)
- **재튜닝**: E_ML contamination 0.05→실불량률(~0.002)
- **보류**: A_TempReg(온도핀 Gage R&R로 신뢰도 확보 후), GP 공간회귀(64bit/직접구현 시)
- **불필요**: dU/dT 보정, 절대온도 아렌니우스 단독, 위치 룩업/트레이평균 패턴, 온도변화량의 "신규 불량인자"화(물리 검증 전)
- **★결정적 미해결**: C/B가 잡은 "Baseline 미검출 추가 ~60셀"이 진짜 누락(미검)인지 과검인지 → **분해/자가방전 재측정 물리 검증** 필요(라벨 순환 깨기). 차선 진단: 트레이별 B_NNR 잔차 vs E등급 — E등급이 NNR잔차로도 튀면 진짜, 정상이면 과검후보
